## 1. 产品概述

### 1.1 一句话定义

AIV 是一个面向 AIGC 视频的一体化平台：**不会的人来上课，学会的人来接单，接到单的人用平台工具完成制作**——类 Fiverr 的双向接单市场，加上课程和 AI 生成工作室，构成站内闭环。

### 1.2 要解决的问题

**商家侧（需求方）：**
- 有大量轻量级视频需求（商品上身展示、开业宣传、口播、Logo 动画等），传统制作公司报价高、周期长；
- 不知道去哪里找「会用 AI 做视频、价格合理」的人；
- 担心线上交易被跳单、被烂尾，缺乏保障。

**创作者侧（供给方）：**
- 想用 AI 视频技能变现，但「学什么、怎么接单、用什么工具」三件事分散在不同平台；
- 学完没有订单来源，接了单工具成本和选型又是门槛；
- 缺乏可信的能力证明，新人冷启动难。

### 1.3 解决方案：三模块内循环

```
   ① 上课（课程中心）
        │  学会技能，拿到结业证书
        ▼
   ② 接单（接单市场）
        │  证书背书 → 承接商家真实订单
        ▼
   ③ 创作（工作室）
        │  代币生成 + 工具优惠，完成交付
        ▼
   进阶课程 ──→ 回到 ①，接更大的单
```

三个模块互相供血：**课程供给技能与信任（证书），市场供给订单与收入，工作室供给产能与耗材收入**。循环每转一圈，创作者更值钱，平台多三层收入。

> 产品表达约定：站内不再使用「飞轮」可视化（V1 已移除），首页仅以低调的「三步，一个循环」条带呈现；但商业逻辑仍按此循环设计。

### 1.4 目标用户

| 角色 | 画像 | 核心诉求 |
| --- | --- | --- |
| 学员/创作者 | 学生、自由职业者、转型中的剪辑师/设计师 | 学会 AI 视频 → 稳定接单变现 |
| 商家 | 淘宝/抖音店主、本地门店、品牌方、内容机构 | 便宜、快、有保障地拿到能用的视频 |
| 讲师 | 实战派创作者/操盘手 | 课程分成收入 + 个人品牌 + 学员转化为自己的客源 |

**关键设定：角色可重叠。** 同一账号既可以作为创作者接单，也可以作为商家发需求（个人主页有双重身份徽章与双视角订单管理）。讲师同时可以是市场里的卖家（讲师主页直接联动其接单服务）。

## 2. 功能需求

### 2.1 首页（参考 Fiverr，保持简洁）

| 编号 | 需求 | 说明 |
| --- | --- | --- |
| H-1 | 搜索式 Hero | 一句标题 + 搜索框 + 热门搜索标签；搜索跳转市场并自动选中命中更多的标签页（需求/服务） |
| H-2 | 三大模块 Banner | 课程中心 / 接单市场 / 工作室 各一张，首页核心位 |
| H-3 | 循环条 | 「三步，一个循环」：① 上课学技能 → ② 市场接真单 → ③ 工作室交付 ↻ 进阶再来一轮；附信任数据（学员数/订单数/担保） |
| H-4 | 内容预览 | 精选课程 ×3、最新商家需求 ×3、优选创作者服务 ×3 |

### 2.2 课程中心

| 编号 | 需求 | 说明 |
| --- | --- | --- |
| C-1 | 课程列表 | 卡片（封面/难度/评分/在学人数/价格），按难度筛选 |
| C-2 | 免费试听 | 每套课**第 1 节永远免费**，未购可直接播放；其余课时锁定 |
| C-3 | 购课解锁 | 一次购买，解锁全部课时 + 学员社区 + 聊天室；演示价 ¥259–599/套 |
| C-4 | 学员社区 | 贴吧式异步讨论板：讲师置顶帖、发帖/回复/点赞；未购显示模糊预览 + 解锁引导 |
| C-5 | 实时聊天室 | 与社区并列的群聊房间：在线人数、即时消息流；购课解锁，未购同样模糊+引导 |
| C-6 | 结业证书 | 完课自动颁发，**带全局序号**（见 2.5）；证书展示在创作者接单主页，作为市场信任背书 |
| C-7 | 讲师主页 | 学员数/课程数/评分/**已颁发证书数**、讲师介绍、开设的课程、**「讲师也在市场接单」**（联动其服务）、学员评价 |
| C-8 | 学完推荐 | 课程侧栏推荐进阶课，驱动循环回流 |

### 2.3 接单市场（双向）

**A. 商家需求（参考 pump.fun/go 的「发布-参与」模式）**

| 编号 | 需求 | 说明 |
| --- | --- | --- |
| M-1 | 发布需求 | 标题/预算区间/期限/分类/详细描述；示例场景：淘宝女装店发布「5 件连衣裙模特上身展示视频」 |
| M-2 | 需求详情 | 需求明细清单、参与人数、状态（报名中/选稿中/进行中） |
| M-3 | 提案机制 | 创作者直接提交报价 + 方案参与；或先站内沟通明确需求再报价 |
| M-4 | 提案详情页 | 完整方案陈述、**分步交付计划**、相关作品集、报价与预计交付侧栏 |
| M-5 | 选用提案 | 商家在自己的需求下可「选用此提案」→ 提案标记已选用，需求转入进行中 |
| M-6 | 商家主页 | 认证标识、发布需求数/成交订单/累计支出/**按时打款率**、在招需求、「创作者这样评价 TA」 |

**B. 创作者服务（参考 Fiverr 的 Gig 模式）**

| 编号 | 需求 | 说明 |

| M-9 | 卖家信任 | 等级（Lv.x）、评分、在线状态、排单数、平均响应时长、**课程技能认证标签** |

### 2.4 工作室（生成 + 工具箱，合并为一个模块）

| 编号 | 需求 | 说明 |
| --- | --- | --- |
| S-1 | 提示词生成视频 | 参考 Seedance 交互：文生视频/图生视频、模型档位（Turbo 快速 / Pro 高质量）、画幅（9:16 / 16:9 / 1:1）、时长（5s/10s）、分辨率（720p/1080p） |

## 3. 商业模式

| 收入线 | 机制 | 对应模块 |
| --- | --- | --- |
| 课程分成 | 课程售价平台抽成（建议 20–30%，讲师拿大头） | 课程中心 |
| 订单佣金 | 担保交易抽佣（建议 5–10%，从打款中扣除） | 接单市场 |
| 代币销售 | 生成服务毛利（代币售价 − 模型 API 成本），大额套餐加赠 | 工作室 |
| 工具返利 | 第三方工具优惠码/返利链接的 CPS 分成（过渡期收入） | 工具箱 |

循环的商业意义：**同一个用户在一圈循环里被变现三次**（买课 → 平台抽佣 → 买代币），且每一次变现都在为下一次创造条件。

## 4. 关键指标

**北极星指标：月度完成订单数（双边市场健康度的最终体现）。**

| 环节 | 指标 |
| --- | --- |
| 课程 | 试听→购课转化率、完课率、证书颁发数 |
| 循环 1→2 | 完课后 30 天内提交首个提案/接到首单的比例 |
| 市场 | 需求填充率（有提案的需求占比）、成交率、纠纷率、复购率 |
| 循环 2→3 | 订单创作者的工作室使用率、单均代币消耗 |
| 循环 3→1 | 接单创作者的进阶课购买率 |
| 商业 | GMV、三线收入占比、创作者月均收入（供给侧留存的根本） |

## 5. 版本范围

### 5.1 已实现（当前前端原型，纯静态演示）

- 首页（搜索 Hero + 三 Banner + 循环条 + 内容预览）
- 课程：列表/详情/试听/购课解锁/社区/实时聊天室（模拟回复）/证书序号/讲师主页
- 市场：双向列表与详情、提案与提案详情页、选用提案、商家主页、发布需求/服务、站内聊天（模拟）、搜索筛选
- 工作室：模拟生成（代币扣费/进度/记录）、充值、工具箱优惠码
- 个人主页全量；中英双语；响应式

> 原型约定：无后端，数据为内存模拟，刷新重置；支付、生成、消息均为演示效果。

### 5.2 不在本期（明确排除）

后台管理系统、真实支付与结算、真实模型 API、账号体系、消息推送、风控审核、移动端 App。

## 7. 风险与对策

| 风险 | 对策 |
| --- | --- |
| 双边冷启动（先有鸡还是先有蛋） | 先以课程聚集供给侧；平台自营/补贴注入首批需求（官方需求池）；讲师自带订单与案例 |
| 跳单（绕过平台交易） | 资金托管 + 担保权益 + 站内沟通提示 + 评价/证书等只在站内累积的信任资产 |
| AI 内容合规与版权 | 生成内容 AI 标识；服装/人像类需求要求商用授权确认；声音克隆需本人授权书（课程中亦做合规教育） |
| 模型成本波动 | 代币定价与模型档位解耦，保留调价空间；多模型供应商备份 |
| 质量参差导致商家流失 | 证书分级 + 接单等级门槛 + 首单小样机制（提案中常见的「先出 1 条小样」沉淀为产品功能） |

## 8. 名词表

| 名词 | 定义 |
| --- | --- |
| 需求（Brief） | 商家发布的视频订单，含预算区间与期限 |
| 提案（Proposal） | 创作者对需求的报价与方案，含交付计划与相关作品 |
| 服务（Gig） | 创作者发布的标准化接单商品，三档套餐 |
| 代币（Token） | 工作室生成视频的计费单位，充值获得 |
| 证书（Certificate） | 完课颁发的技能认证，带全局序号，展示于接单主页 |
| 双重身份 | 同一账号同时具备创作者与商家角色 |
